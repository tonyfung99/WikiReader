# Mermaid Diagram Rendering & Text Selection Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Render `` ```mermaid `` fenced blocks as real diagrams (with a graceful raw-source fallback), and make rendered note/answer content selectable for copy/share.

**Architecture:** Adds the project's first two third-party dependencies via SPM: `beautiful-mermaid-swift` (app target, pure-Swift native diagram rendering, no WebView) and `ViewInspector` (test target only, structural SwiftUI view testing). A new `MermaidBlockView` hooks into `MarkdownView`'s existing `.code(language:code:)` case by language tag — no parser changes. Text selection is one modifier applied at two root points.

**Tech Stack:** Swift / SwiftUI (iOS 26, Xcode 26), Swift Testing, SPM. Spec: `docs/superpowers/specs/2026-07-16-mermaid-and-text-selection-design.md`.

## Global Constraints

### Task 1: Add SPM dependencies

Adding a remote SPM package via the `xcodeproj` gem is more error-prone than the file-registration scripting used elsewhere in this project — it means constructing `XCRemoteSwiftPackageReference` / `XCSwiftPackageProductDependency` / `PBXBuildFile` objects by hand, not a one-line helper. - [ ] **Step 1: Write and run the package-addition script**

Write this to a temporary file (e.g. `/tmp/add_packages.rb`) — it's a one-off migration script, not something that needs to live in the repo:

```ruby
require 'xcodeproj'

project_path = 'WikiReader.xcodeproj'
project = Xcodeproj::Project.open(project_path)

app_target = project.targets.find { |t| t.name == 'WikiReader' }
test_target = project.targets.find { |t| t.name == 'WikiReaderTests' }
raise "WikiReader target not found" unless app_target
raise "WikiReaderTests target not found" unless test_target

# beautiful-mermaid-swift -> app target only
mermaid_ref = project.new(Xcodeproj::Project::Object::XCRemoteSwiftPackageReference)
mermaid_ref.repositoryURL = 'https://github.com/lukilabs/beautiful-mermaid-swift'
mermaid_ref.requirement = { 'kind' => 'upToNextMajorVersion', 'minimumVersion' => '1.0.0' }
project.root_object.package_references << mermaid_ref

mermaid_product = project.new(Xcodeproj::Project::Object::XCSwiftPackageProductDependency)
mermaid_product.package = mermaid_ref
mermaid_product.product_name = 'BeautifulMermaid'
app_target.package_product_dependencies << mermaid_product

mermaid_build_file = project.new(Xcodeproj::Project::Object::PBXBuildFile)
mermaid_build_file.product_ref = mermaid_product
app_target.frameworks_build_phase.files << mermaid_build_file

# ViewInspector -> test target only, never the app target
inspector_ref = project.new(Xcodeproj::Project::Object::XCRemoteSwiftPackageReference)
inspector_ref.repositoryURL = 'https://github.com/nalexn/ViewInspector'
inspector_ref.requirement = { 'kind' => 'upToNextMajorVersion', 'minimumVersion' => '0.9.0' }
project.root_object.package_references << inspector_ref

inspector_product = project.new(Xcodeproj::Project::Object::XCSwiftPackageProductDependency)
inspector_product.package = inspector_ref
inspector_product.product_name = 'ViewInspector'
test_target.package_product_dependencies << inspector_product

inspector_build_file = project.new(Xcodeproj::Project::Object::PBXBuildFile)
inspector_build_file.product_ref = inspector_product
test_target.frameworks_build_phase.files << inspector_build_file

project.save
puts "Added beautiful-mermaid-swift to #{app_target.name}, ViewInspector to #{test_target.name}"
```

Expected output: `Added beautiful-mermaid-swift to WikiReader, ViewInspector to WikiReaderTests`

### Task 4: Mermaid rendering — `MermaidBlockView`

Renders `` ```mermaid `` fenced blocks as diagrams, with a graceful fallback to the raw source on parse failure. Split into three types across one new file: `MermaidBlockView` (owns the live `@State`/bindings the library drives), `MermaidBlockContentView` (the branching decision as a pure function of its inputs — testable and previewable without a live render pass, matching the project's `ShareStatusView(phase:)`-style "presentation is a function of a value" convention), and `MermaidFallbackView` (the raw-source fallback, also directly testable). All three are `internal` (no `private`), not the file's usual `private`-subview convention — required so `WikiReaderTests` can reference `MermaidBlockContentView` and `MermaidFallbackView` directly via `@testable import`.

```swift
import SwiftUI
import BeautifulMermaid

/// Renders a fenced ```mermaid``` code block as a native diagram, falling
/// back to the raw source (never a crash or blank space) when the syntax
/// is malformed or uses a diagram type the library doesn't support.
struct MermaidBlockView: View {
    let source: String

    @Environment(\.colorScheme) private var colorScheme
    @State private var parseError: Error?
    @State private var diagramBounds: CGRect = .zero

    var body: some View {
        MermaidBlockContentView(
            source: source,
            errorMessage: parseError?.localizedDescription,
            diagramBounds: diagramBounds,
            theme: colorScheme == .dark ? .githubDark : .githubLight,
            parseErrorBinding: $parseError,
            diagramBoundsBinding: $diagramBounds
        )
    }
}

/// The rendering decision as a pure function of its inputs — testable and
/// previewable without a live MermaidDiagramView render pass. A non-nil
/// `errorMessage` selects the fallback branch; nil selects the diagram.
struct MermaidBlockContentView: View {
    let source: String
    let errorMessage: String?
    let diagramBounds: CGRect
    let theme: DiagramTheme
    @Binding var parseErrorBinding: Error?
    @Binding var diagramBoundsBinding: CGRect

    var body: some View {
        if let errorMessage {
            MermaidFallbackView(source: source, message: errorMessage)
        } else {
            ScrollView(.horizontal, showsIndicators: false) {
                MermaidDiagramView(
                    source: source,
                    theme: theme,
                    parseError: $parseErrorBinding,
                    diagramBounds: $diagramBoundsBinding
                )
                .frame(
                    width: diagramBounds.width > 0 ? diagramBounds.width : 300,
                    height: diagramBounds.height > 0 ? diagramBounds.height : 200
                )
            }
        }
    }
}

/// Raw-source fallback for a Mermaid block that failed to parse or uses an
/// unsupported diagram type.
struct MermaidFallbackView: View {
    let source: String
    let message: String

    var body: some View {
        VStack(alignment: .leading, spacing: 6) {
            Label("Diagram unavailable", systemImage: "exclamationmark.triangle")
                .font(.footnote)
                .foregroundStyle(.secondary)
            ScrollView(.horizontal, showsIndicators: false) {
                Text(source)
                    .font(.system(.callout, design: .monospaced))
                    .padding(12)
            }
            .background(Color.secondary.opacity(0.12))
            .clipShape(RoundedRectangle(cornerRadius: 8))
        }
    }
}

#Preview("Diagram") {
    ScrollView {
        MermaidBlockView(source: """
        graph TD
            A[Start] --> B{Decision}
            B -->|Yes| C[Do Something]
            B -->|No| D[Do Something Else]
            C --> E[End]
            D --> E
        """)
        .padding()
    }
}

#Preview("Fallback") {
    MermaidBlockContentView(
        source: "pie title Unsupported\n  \"A\" : 1",
        errorMessage: "Unsupported diagram type.",
        diagramBounds: .zero,
        theme: .githubLight,
        parseErrorBinding: .constant(nil),
        diagramBoundsBinding: .constant(.zero)
    )
    .padding()
}
```
